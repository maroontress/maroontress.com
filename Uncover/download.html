必要な構成

ビルドできたプラットフォーム

以下のプラットフォームでビルドを確認しました（i386アーキテクチャのみ）。

  • FreeBSD 8.0
  • Ubuntu 9.10
  • openSUSE 11.2

ビルドに必要なもの

ビルドは次のものに依存しています。

  • Apache Ant 1.7.1
  • Java SE Development Kit（JDK）6

各プラットフォームではそれぞれ次のように準備するのが簡単です。

FreeBSD 8.0
以下のportsをインストールしておきます。
  • java/jdk16
  • devel/apache-ant
Ubuntu 9.10
次のパッケージをインストールしておきます。
  • sun-java6-jdk
  • ant
  • ant-optional
openSUSE 11.2
次のRPMをインストールしておきます。
  • java-1_6_0-sun-devel-1.6.0.u20-0.1.1
  • ant-1.7.1-7.1.noarch
  • ant-nodeps-1.7.1-7.1.noarch

実行に必要なもの

実行するには次のものをインストールしておく必要があります。

  • Java SE Runtime Environment (JRE) 6
  • SQLite
  • SQLite JDBCドライバ

各プラットフォームではそれぞれ次のように準備するのが簡単です。

FreeBSD 8.0
以下のportsをインストールしておきます。
  • java/jdk16
  • java/sqlitejdbc
  • database/sqlite3
Ubuntu 9.10
http://www.xerial.org/trac/Xerial/wiki/SQLiteJDBCからjdbc-3.6.20.1.jarを取得しておきます。加えて、次のパッケージをインストールしておきます。
  • sun-java6-jre
  • sqlite3
openSUSE 11.2
http://www.xerial.org/trac/Xerial/wiki/SQLiteJDBCからjdbc-3.6.20.1.jarを取得しておきます。加えて、次のRPMをインストールしておきます。
  • java-1_6_0-sun-1.6.0.u20-0.1.1
  • sqlite3-3.6.16-2.3.i586

ビルドとインストール

次の手順でビルド、インストールします。

% tar jxf uncover-YYYYMMDD.tar.bz2
% cd uncover-YYYYMMDD/
% ant
% su
# ant -Dprefix=path install ★1

★1 pathには絶対パス（/usr/localや/home/fooなど）を指定します。インストールされるファイルは次の通りです。

path/bin/uncover
path/lib/uncover/jdbcskirt.jar
path/lib/uncover/uncover.jar

設定

環境変数UNCOVER_JVM_OPTIONS

JVM（java）に渡したいオプションがある場合は、環境変数UNCOVER_JVM_OPTIONSに設定しておきます。

SQLite JDBCドライバのパスの設定

configコマンドでSQLite JDBCドライバ（jarファイル）があるディレクトリのパスを設定します。例えば、/usr/local/share/java/classes/にJDBCドライバをおいた場合は、次のように指定します。

% uncover config ext.dirs /usr/local/share/java/classes

デフォルトのデータベースファイルのパスの設定

configコマンドでデフォルトのデータベースファイルのパスを設定します。例えば、$HOME/uncover.dbをデフォルトのデータベースファイルとする場合は、次のように設定します。

% uncover config db.default $HOME/uncover.db

SQLiteを使用するため、データベースファイルはNFS上におかないでください。

以降、Uncoverでデータベースにアクセスするコマンドを実行するときは、コマンドラインオプション（--db=pathまたは-d path）でデータベースファイルを指定しない場合、データベースファイルに$HOME/uncover.dbを使用するようになります。

設定の確認

configコマンドで設定した値をshow-configコマンドで次のように確認できます。

% uncover show-config
...

基本的な使い方

データベースの作成

次のようにinitコマンドを使用して、データベースを作成します。

% uncover init

initコマンドに限らず、デフォルトのパス以外のデータベースファイルを使用する場合は、--dbまたは-dオプションを使用してください。例えば、$HOME/foo.dbをデータベースファイルのパスとする場合は、次のように作成します。

% uncover --db=$HOME/foo.db init

または、次のように指定することもできます。

% uncover -d ~/foo.db init

カバレッジとカバレッジ結果の登録

測定対象のプロジェクトをビルド、テストを実行して、gcno/gcdaファイルを作成します。例として、fooプロジェクトのリビジョン1.0を次のようにカバレッジします。

% tar zxf foo-1.0.tar.gz
% cd foo-1.0/
% ./configure CFLAGS="-O0 -ftest-coverage -fprofile-arcs" --disable-shared
...
% make all check
...

続けて、gcno/gcdaファイルを次のようにUncoverのデータベースに登録します。

% find . -name "*.gcno" \
  | uncover commit-note --input-file=- --project=foo --revision=1.0
...

データベースの登録の確認

次のように、プロジェクトとリビジョンを登録できたかどうか確認します。

% uncover list-projects
foo
% uncover list-revisions --project=foo
1.0

次のカバレッジ結果の登録

同様に、fooプロジェクトのリビジョン1.1をカバレッジしたあと、次のようにカバレッジ結果をデータベースに登録します。

% tar zxf foo-1.1.tar.gz
% cd foo-1.1/
% ./configure CFLAGS="-O0 -ftest-coverage -fprofile-arcs" --disable-shared
...
% make all check
...
% find . -name "*.gcno" \
  | uncover commit-note --input-file=- --project=foo --revision=1.1
...

再び登録したリビジョンの確認

次のように、fooプロジェクトのリビジョン1.0と1.1がデータベースに登録されたことを確認します。 

% uncover list-revisions --project=foo
1.0
1.1

レポートの出力

次のように、リビジョン1.0と1.1の比較レポートをHTML形式で出力します。

% uncover report --project=foo 1.0 1.1 > foo-1.0--1.1.html

高度な使い方

リビジョンの削除

プロジェクトfooに登録されているリビジョン1.0をデータベースから削除するには、次のようにします。

% uncover delete-revision --project=foo 1.0

プロジェクトの削除

プロジェクトfooをデータベースから削除するには、次のようにします。

% uncover delete-project foo

新規プロジェクトの作成

空のプロジェクトfooをデータベースに作成するには、次のようにします。

% uncover new-project foo

Covertureが出力したXMLファイルの登録

Covertureでgcno/gcdaファイルをXMLファイルに変換し、それをデータベースに登録するには、次のようにします。

% find . -name "*.gcno" | coverture --input-file=- > foo-1.0.xml
...
% uncover commit --project=foo --revision=1.0 foo-1.0.xml

関数グラフの出力

関数グラフをgraphvizで処理できる形式で出力することができます。例えば、プロジェクトfoo、リビジョン1.0のファイルBar.cにある関数baz()の関数グラフを出力するには、次のようにします。

% uncover graph --project=foo --revision=1.0 baz@Bar.gcno \
  > foo-1.0-baz@Bar.dot

生成したdotファイルをpngファイルに変換するには、次のようにします。

% dot -Tsvg foo-1.0-baz@Bar.dot | svg2png > foo-1.0-baz@Bar.png
